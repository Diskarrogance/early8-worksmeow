# 自进化机制

> 参考 [PI Agent](https://github.com/earendil-works/pi-agent) 设计理念：Agent 的核心特征不是"智能"，而是**能修改自身**。

---

## 核心理念

PI Agent 的系统提示词 < 1000 tokens，Agent 可以**读写自身配置文件并热加载**。这意味着：

```
Agent 发现缺陷 → 修改自身 prompt → 下次调用即生效
                                   ↑ 不需要重启、不需要部署
```

---

## 实现路径

### 第一阶段：配置可写 (MVP)

Agent 可通过工具方法读取和写入自身配置：

```
readSelfConfig()    → 返回当前 prompt / 模型参数 / 工具列表
writeSelfConfig()   → 修改配置并触发热加载
```

安全边界：
- **修改 prompt / model / temperature** → 自动备份原配置到 `.backup` 目录
- **新增工具** → 白名单检查
- **删除工具** → 手动确认
- **修改系统文件** → 手动确认

### 第二阶段：行为日志驱动自我优化 (v2)

```
Agent 执行任务
  └→ 记录执行日志（耗时、错误、工具调用）
      └→ 定期自检（每天一次 / 按事件触发）
          └→ 分析日志模式
              ├→ 发现常见错误 → 更新 prompt 增加处理规则
              ├→ 发现低效路径 → 优化工具调用顺序
              └→ 发现新需求 → 建议添加新工具/Skill
```

### 第三阶段：元认知 (v3)

```
Agent 不仅有知识，还有"关于自己如何思考的知识"
  ├→ 监控自身上下文窗口使用率
  ├→ 注意到某些工具被高频调用 → 生成快捷方式
  └→ 注意到某些问题反复出现 → 创建新的子 Agent
```

---

## 安全边界

| 操作 | 默认策略 | 用户干预 |
|------|---------|---------|
| 修改自身 prompt | ✅ 自动（备份原文件） | 可配置通知 |
| 修改模型参数 | ✅ 自动（备份） | 可配置通知 |
| 添加新工具 | ⚠️ 白名单通过则自动 | 白名单外须确认 |
| 删除工具 | ❌ 须用户确认 | 弹窗确认 |
| 写文件系统 | ❌ 须用户确认 | 弹窗确认 |
| 执行 Shell | ❌ 须用户确认 | 弹窗确认（可配置白名单） |

---

## 配置示例

```json
{
  "selfEvolution": {
    "enabled": true,
    "backupDir": "config/.backup",
    "autoConfirm": {
      "promptModify": true,
      "modelParamModify": true,
      "toolAddWhiteList": ["read", "write", "bash"]
    },
    "schedule": {
      "selfReview": "every 24h"
    }
  }
}
```
